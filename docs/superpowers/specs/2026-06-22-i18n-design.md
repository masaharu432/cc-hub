# cc-hub i18n（英語デフォルト + 日本語/英語切替）設計

- 日付: 2026-06-22
- 対象: `web/index.html`, `web/app.js`, `web/i18n.js`（新規）, `server.py`, `README.md`, `README.ja.md`

## 背景・目的

cc-hub は現状 UI もエラーメッセージもハードコードされた日本語のみ。公開リポジトリ（GitHub）として世界向けに出すため、

1. README を**英語をデフォルト**にする（日本語版は併存）。
2. アプリ UI を**英語・日本語の双方**で表示でき、**設定から切り替え**られるようにする。

を実現する。会話そのものは公式アプリが担うため、本対応は cc-hub の管理 UI に閉じる。

## 決定事項（ユーザー確認済み）

- **初期言語**: `navigator.language` を自動判定し、日本語端末なら日本語 UI、それ以外は英語にフォールバック。`localStorage.lang` があればそれを優先。
- **翻訳スコープ**: フロントエンド（`index.html` + `app.js`）を全面 i18n 化（en/ja）。`server.py` のユーザー向け日本語メッセージは**英語の単一言語に書き換える**（i18n 化せず、言語トグルには追従しない）。
- **切替 UI**: ヘッダのギア（⚙）アイコン → 既存のフォルダ参照と同じ Bootstrap offcanvas で開く**設定パネル**。中身は当面「言語（English / 日本語）」のみ。将来拡張の器として用意するが、現時点で他項目は追加しない（YAGNI）。
- **実装方式**: 辞書 + `t()` 関数方式（下記「実装方式」）。

## 実装方式（採用 = A 案）

1 つの文字列辞書を全 UI で共有する。サーバ往復なし、完全クライアントサイド。

- `I18N = { en: {...}, ja: {...} }` をキーで引く。
- 静的 HTML は `data-i18n` 属性で宣言し、`applyI18n()` が DOM を一掃して流し込む。
- 動的 JS は日本語リテラルを `t(key, params)` 呼び出しに置換する。

却下案:
- **B 案（言語別 HTML / 文字列テーブルのファイル分割で丸ごと差替え）**: 重複が増えメンテ性が悪い。
- **C 案（両言語を DOM に出し CSS `[lang]` で出し分け）**: 静的部分は楽だが `app.js` が生成する動的 UI に弱く DOM が膨らむ。

## コンポーネント構成

### `web/i18n.js`（新規・`app.js` より前に読み込む）

公開 API:

- `getLang()` → 現在言語（`'en'` | `'ja'`）。決定順: `localStorage.lang` → `navigator.language` が `ja*` なら `'ja'` → それ以外 `'en'`。
- `setLang(lang)` → `localStorage.lang` に保存し、`<html lang>` を更新。再描画は呼び出し側が行う。
- `t(key, params)` → 現在言語の辞書から文字列を引き、`{name}` 形式のプレースホルダを `params` で補間。キー欠落時は `key` をそのまま返す（壊さない）。
- `applyI18n(root = document)` → `root` 配下の `data-i18n` 要素を走査して文言を流し込む。

`data-i18n` の属性仕様:

- `data-i18n="key"` → `textContent` を設定。
- `data-i18n-attr="placeholder:key; title:key2"` → 指定属性に設定（複数はセミコロン区切り）。
- `data-i18n-html="key"`（必要時のみ・限定使用）→ `innerHTML`。原則使わない。

辞書のキー設計: 画面領域でグルーピングする。例 `nav.projects`, `auth.token_label`, `launch.title`, `launch.btn`, `sessions.adopt_all`, `search.placeholder`, `archive.restore_hint`, `settings.language`, `toast.launched`, `swipe.rename`, `swipe.archive` 等。

### `web/index.html`

- 全ユーザー向け文言を `data-i18n` 属性化（約 39 箇所）。
- ヘッダ（ナビ付近）に**ギアボタン**を追加し、設定 offcanvas を開く。
- 設定 offcanvas（フォルダ参照 offcanvas と同じパターン）を追加。中身は言語選択（English / 日本語 のラジオまたはセグメント）。
- `<html lang="ja">` の初期値は `i18n.js` が起動時に上書きするため固定値依存をなくす。
- `<script src="/i18n.js">` を `app.js` より前に読み込む。

### `web/app.js`

- ハードコード日本語リテラル（約 86 箇所: セッション一覧、トースト、スワイプアクション、確認ダイアログ、検索・アーカイブ・spawn 各ビューの動的生成文言）を `t()` に置換。
- 設定パネルの言語切替ハンドラ: `setLang(l)` → `applyI18n(document)` → **現在アクティブなビューの既存 render/refresh を呼び直す**（動的 DOM を再生成して即時反映）。
- 初期化フロー: ページ読込 → `applyI18n(document)` → 既存の初期描画。

### `server.py`

- ユーザーに返る日本語 `ValueError` / エラーメッセージ（約 12 箇所、例: `稼働中のセッションです。…` / `reptyr が使えません…` / `会話ログが見つかりません。` / `(フォルダ不明)` など）を**英語に書き換え**。
- ソースコメント中の日本語はそのまま（ユーザーに出ない）。
- 正規表現の許可文字（日本語を許す挙動）など機能ロジックは変更しない。

## データフロー

```
ページ読込
  → getLang() で言語決定（localStorage.lang ?? navigator.language→ja/en）
  → <html lang> 設定
  → applyI18n(document)（静的文言）
  → app.js 初期描画（t() で動的文言）

言語切替（設定パネル）
  → setLang(l)
  → applyI18n(document)
  → アクティブビューを再描画
```

サーバ往復は一切なし。

## README

- `README.md` を**英語（デフォルト）**に新規翻訳。
- 現行の日本語版 README を `README.ja.md` に退避（内容は現行のまま）。
- 両ファイル冒頭に言語リンク（`English | 日本語`）。
- 画像参照（`docs/images/*.png`）は両言語共通でそのまま流用。
- 公開ミラー（cc-hub-public）にこの構成がそのまま乗る前提。

## テスト / 検証

- **キー整合チェック**: `I18N.en` と `I18N.ja` のキー集合が一致すること（欠けキー検出）。簡易チェックで担保。
- **目視（実機相当）**:
  - 言語トグルで静的 + 動的（セッション一覧・トースト・スワイプアクション・検索・アーカイブ・設定）が即時に切り替わる。
  - リロード後も選択言語が保持される。
  - `localStorage.lang` 未設定時、日本語端末で日本語・それ以外で英語になる。
  - サーバエラー（例: 稼働中セッションのアーカイブ拒否）が英語で表示される。

## スコープ外（YAGNI）

- server.py エラーの言語トグル追従（英語単一で固定）。
- 設定パネルの言語以外の項目。
- 3 言語目以降。ただしキー辞書方式なので追加は容易。
